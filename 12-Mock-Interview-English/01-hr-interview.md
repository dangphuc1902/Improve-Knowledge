# PART 1 — HR Interview Questions
> Tailored to your CV: Gihot Studio → Hahalolo → FPM Project

---

## ❓ Q1: "Tell me about yourself."

### 🎯 Interviewer's Intention
- Can you communicate clearly in English?
- Do you understand your own career story?
- Are you the right fit for this role?

### ✅ Sample Answer

> "My name is Phuc, and I'm a Backend Developer with over 3 years of experience, primarily in Java and C++. I started my career at Hahalolo — a travel social network — where I built RESTful APIs using Java 8 and Spring Boot, and implemented Spring Security with JWT for role-based access control. After that, I joined Gihot Studio, a game company, where I moved into a more specialized area: building high-performance game servers in C++17. There, I designed real-time multiplayer systems supporting over 5,000 concurrent users, and I also built Spring Boot microservices for admin and monitoring tools.
>
> Outside work, I've been building a personal project called FPM — a Financial Portfolio Manager — which is a 10-service cloud-native system using Java 21, Spring Cloud, Kafka, RabbitMQ, and gRPC. It's been my way of deepening my knowledge of distributed systems and microservices architecture.
>
> I'm now looking for a new challenge at a company where I can apply my Java backend expertise at a larger, enterprise scale."

### 🇻🇳 Vietnamese Explanation
- Bắt đầu bằng tên và số năm kinh nghiệm
- Kể theo thứ tự thời gian: Hahalolo → Gihot → FPM
- Kết bằng lý do muốn tham gia công ty mới (forward-looking)
- Không kể quá 2 phút — bài này dài ~1:30

### ❌ Common Mistakes
- Đọc CV verbatim thay vì kể chuyện
- Quá dài (>3 phút)
- Không nhắc đến Java khi JD yêu cầu Java
- Kết bằng câu mơ hồ "I want to grow my career"

## ❓ Q2: "Why do you want to leave your current company?"

### 🎯 Interviewer's Intention
- Are you leaving for positive reasons or red flags?
- Are you a job-hopper?
- Will you stay long-term?

### ✅ Sample Answer

> "Gihot Studio has been a great place to learn. I've grown a lot in terms of C++ game server development and distributed systems. However, the core product focus is on game backend in C++, and I want to deepen my Java backend expertise — particularly in enterprise and business domains like fintech or telecom. I feel that this role aligns better with that direction. I'm looking for an environment where Java is the primary language and where I can work on large-scale business systems with complex requirements."

### 🇻🇳 Vietnamese Explanation
- Không bao giờ nói xấu công ty cũ
- Nhấn mạnh lý do "hướng tới" (muốn học Java enterprise) thay vì "chạy khỏi" công ty cũ
- Mention rằng bạn muốn apply Java skill vào business domain

### ❌ Common Mistakes
- "My boss is difficult" — tuyệt đối không nói
- "The salary is low" — có thể nói nhẹ nhàng nhưng không nên là lý do duy nhất
- Nói xấu đồng nghiệp

## ❓ Q3: "Why should we hire you?"

### 🎯 Interviewer's Intention
- Can you articulate your value clearly?
- Do you understand what the job needs?
- Are you confident without being arrogant?

### ✅ Sample Answer

> "I believe I bring three key things to this role.
>
> First, **proven backend experience**: I've built production systems that serve real users — from 5,000 concurrent game sessions at Gihot to RESTful APIs for a travel social network at Hahalolo.
>
> Second, **microservices expertise**: My personal project, FPM, is a 10-service system using Spring Boot 3, Spring Cloud, Kafka, gRPC, and Resilience4j — all technologies relevant to modern Java backend development.
>
> Third, **performance mindset**: I've consistently optimized systems — 40% SQL query reduction at Hahalolo, 30% DB load reduction with Redis caching at Gihot, and 25% latency improvement under 10,000 concurrent user load tests.
>
> I'm confident I can contribute from day one and continue to grow with your team."

### 🇻🇳 Vietnamese Explanation
- Dùng cấu trúc "3 điểm mạnh" — dễ nhớ, rõ ràng
- Mỗi điểm phải có **số liệu cụ thể** từ CV của bạn
- Kết bằng sự tự tin nhưng humble

### ❌ Common Mistakes
- "I'm a hard worker" — quá chung chung
- Không link điểm mạnh với JD
- Liệt kê kỹ năng mà không có bằng chứng

## ❓ Q4: "What are your strengths?"

### 🎯 Interviewer's Intention
- Self-awareness
- Relevance to the role

### ✅ Sample Answer

> "My strongest areas are **performance optimization** and **system design thinking**. I approach every feature with scalability in mind from the start. For example, at Gihot, when we were seeing latency spikes under load, I profiled the system, identified Redis cache misses as the bottleneck, and implemented a TTL caching strategy that reduced DB load by 30%. I don't just write code that works — I write code that works *well under pressure*."

### 🇻🇳 Vietnamese Explanation

- Luôn có ví dụ cụ thể (STAR format)
- Tránh nói "I'm a fast learner" mà không có bằng chứng

### ❌ Common Mistakes
- Nói quá nhiều điểm mạnh (>3)
- Điểm mạnh không liên quan đến job
- "I'm very hardworking and dedicated" — quá cliché

## ❓ Q5: "What are your weaknesses?"

### 🎯 Interviewer's Intention
- Self-awareness và honesty
- Are you working on it?

### ✅ Sample Answer

> "Honestly, one area I'm actively working on is **English communication**. I'm comfortable reading technical documentation and writing in English, but when speaking in fast-paced meetings, I sometimes slow down to find the right word. I've been aware of this for a while — that's actually why I've been practicing technical English specifically, studying common phrases for presenting project status, explaining trade-offs, and handling Q&A sessions. I've made real progress over the past few months."

### 🇻🇳 Vietnamese Explanation
- Chọn weakness **thật** nhưng **không fatal** với job
- English là weakness hợp lý vì JD yêu cầu TOEIC/IELTS — cho thấy bạn tự nhận thức
- Luôn kèm "I'm working on it" với action cụ thể

### ❌ Common Mistakes
- "My weakness is I work too hard" — sáo rỗng, interviewer ghét câu này
- Weakness là skill cốt lõi của job (VD: "I don't know Spring Boot")
- Không có hành động cải thiện

## ❓ Q6: "What are your salary expectations?"

### 🎯 Interviewer's Intention
- Are you in the budget range?
- Can you negotiate professionally?

### ✅ Sample Answer (Mid-Senior)

> "Based on my experience and research on the market, I'm looking for a range of **25–35 million VND gross** per month. That said, I'm open to discussion — I'd love to understand the full compensation package, including benefits and growth opportunities, before settling on a specific number."

### 🇻🇳 Vietnamese Explanation
- Luôn đưa ra **range** (không phải 1 số cố định)
- Nói "gross" để rõ ràng
- Mở ra thảo luận thay vì chốt cứng
- Research mức lương thị trường trước (ITViec, TopDev)

### ❌ Common Mistakes
- "I'll accept whatever you offer" — mất leverage
- Đưa số quá cao ngay từ đầu mà không có justify
- Không đề cập đến benefits

## ❓ Q7: "Describe a challenge you faced."

### 🎯 Interviewer's Intention
- Problem-solving ability
- How do you handle pressure?

### ✅ Sample Answer (STAR Format)

> "**Situation**: At Gihot Studio, during stress testing for our card battle game, we discovered that at 5,000 concurrent users, we were seeing race conditions in the player state management system. Players would sometimes see incorrect scores or receive duplicate rewards.
>
> **Task**: I was responsible for identifying and fixing this before the launch deadline — which was 2 weeks away.
>
> **Action**: I started by reproducing the bug systematically using load tests. I traced the issue to a non-atomic read-modify-write pattern in the reward distribution code. I redesigned the critical section using mutex locks and atomic operations in C++17. I also added a Redis-based idempotency key to prevent duplicate reward processing at the distributed level.
>
> **Result**: After the fix, we ran 10,000 concurrent user stress tests with zero race conditions detected. We launched on time, and the game ran stable in production."

### ❌ Common Mistakes
- Vague situation ("We had some performance problems")
- No clear personal contribution ("We fixed it as a team")
- No measurable result

## ❓ Q8: "Describe a conflict in your team."

### ✅ Sample Answer

> "At Hahalolo, there was a disagreement between me and a senior developer about database design. He wanted to denormalize tables for performance, while I argued that maintaining normalization with proper indexing would be more maintainable long-term. Rather than escalating it, I proposed we run a benchmark — I implemented a POC for both approaches and ran realistic load tests. The results showed that with the right indexes, the normalized approach was actually 15% faster for our read patterns. We went with my approach, and it resolved the conflict based on data rather than opinion."

### 🇻🇳 Vietnamese Explanation
- Không đổ lỗi cho ai
- Focus vào quá trình giải quyết (data-driven)
- Show rằng bạn có thể handle disagreement một cách professional

---

## ❓ Q9: "Describe a failure."

### ✅ Sample Answer

> "Early in my career at Hahalolo, I pushed a SQL query optimization to production without adequate load testing. The query looked fine in development, but under real traffic, it caused table-lock issues and degraded API response time by over 50% for about 20 minutes until we rolled back. It was a significant incident. I took full responsibility, wrote a postmortem document, and proposed a new rule: all DB changes must go through a load test in a staging environment that mirrors production. That policy was adopted by the team and we haven't had a similar incident since."

### ❌ Common Mistakes
- "I can't think of a failure" — sounds dishonest
- Failure where you blame others
- No lesson learned / no action taken

## ❓ Q10: "Describe a successful project."

### ✅ Sample Answer

> "My most comprehensive project is **FPM — Financial Portfolio Manager** — a personal microservices system I designed and built from scratch. It has 10 Spring Boot services covering user authentication, wallet management, transactions, reporting, notifications, OCR, and AI-powered categorization.
>
> The architecture uses Java 21, Spring Cloud, dual-broker messaging (Kafka for high-throughput transaction events, RabbitMQ for domain routing), and gRPC for inter-service communication with Protobuf contracts. I implemented defense-in-depth security: dual-layer JWT validation at gateway and service level, Redis token blacklisting, and row-level ownership checks.
>
> The most rewarding part was designing 9 shared internal libraries to standardize error handling, JWT, and event publishing across all services. This reduced code duplication significantly and made the system maintainable as it grew.
>
> I'm proud of it because it demonstrates my ability to design production-grade distributed systems — not just write CRUD endpoints."
