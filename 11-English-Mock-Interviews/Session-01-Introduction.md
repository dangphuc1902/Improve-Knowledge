# SESSION 01 — Self-Introduction & Interview Opener

> **SRS Level**: Foundational | **Review**: Daily for first week
> **Goal**: Sound confident and natural from the very first sentence

---

## 🃏 CARD 1 — Opener Phrases

**Q: The interviewer says "Please introduce yourself."**

❌ Don't say:
- "My name is Phuc and I am Java developer with 3 year experience."
- "I am working in my current company as backend developer."

✅ Do say:
> "Thanks for having me today. I'm Phuc, a Java Backend Engineer with about three years of experience building enterprise applications — primarily in the fintech and telecom space. I've been working with Spring Boot, Hibernate, and Oracle as my core stack, and I've recently been deepening my expertise in microservices with Spring Cloud. I'm genuinely excited about this opportunity because [specific reason]. Where would you like me to start?"

**🔑 Key phrases:**
- *"Thanks for having me"* — Professional, warm opener
- *"about three years"* — sounds natural (not exactly 3)
- *"primarily in"* — means "mainly in"
- *"deepening my expertise"* — better than "learning more"
- *"genuinely excited"* — sounds authentic

## 🃏 CARD 2 — Talking About Your Current Role

**Q: "What do you currently do?"**

✅ Model answer:
> "Currently I'm a Java Backend Engineer at [company]. Day-to-day, I'm responsible for designing and maintaining RESTful APIs that handle about [X] requests per day. I work closely with the frontend team, QA engineers, and sometimes directly with clients to clarify requirements.

> One of the more interesting things I've been working on recently is migrating parts of our monolith to microservices — which has given me hands-on experience with Kafka, Docker, and Spring Cloud."

**🔑 Key phrases:**
- *"Day-to-day"* = on a daily basis, in my daily work
- *"I'm responsible for"* = my role includes
- *"work closely with"* = collaborate với
- *"hands-on experience"* = practical, real experience (not just theory)

**📝 YOUR SCORE: [ ]✅ [ ]⚠️ [ ]❌**

## 🃏 CARD 3 — Describing Your Tech Stack Naturally

**Q: "What technologies do you work with?"**

❌ Robotic answer:
> "I use Java, Spring Boot, Spring MVC, Hibernate, Oracle, Git, Maven."

✅ Natural answer:
> "My primary stack is Java with Spring Boot on the backend. For data persistence, I work with Hibernate and Oracle — we have some pretty complex reporting queries, so I've gotten comfortable with Oracle-specific features like execution plans and stored procedures.

> For the infrastructure side, we deploy on Tomcat and we've recently started containerizing with Docker. I use Maven for builds, Git for version control, and we have a Jenkins pipeline for CI/CD.

> On the web services side, I've built both REST APIs and consumed SOAP services from legacy banking systems — so I'm comfortable with both worlds."

**🔑 Key phrases:**
- *"my primary stack"* = công nghệ chính tôi dùng
- *"gotten comfortable with"* = đã quen thuộc với
- *"on the infrastructure side"* = về mặt hạ tầng
- *"both worlds"* = cả hai thế giới (REST và SOAP)

**📝 YOUR SCORE: [ ]✅ [ ]⚠️ [ ]❌**

## 🃏 CARD 4 — Talking About a Project

**Q: "Tell me about your most significant project."**

✅ Model answer (STAR format):
> "The project I'm most proud of is a digital loan origination system we built for a fintech client.

> The challenge was significant: we needed to process loan applications in real-time, integrate with three external credit bureaus, and maintain full audit trails for regulatory compliance — all while handling 5,000 concurrent users.

> My role was Lead Backend Developer. I designed the microservices architecture — we used Spring Boot for each service, Kafka for async communication between them, Redis for caching credit scores, and Oracle for the transactional database.

> What I'm particularly proud of is that we delivered on schedule, the system processed over 1,000 loan applications on the first day without issues, and the client extended our contract by two years. That was a really satisfying outcome."

**🔑 Key phrases:**
- *"I'm most proud of"* = tôi tự hào nhất về
- *"The challenge was significant"* = Thử thách rất lớn
- *"particularly proud of"* = đặc biệt tự hào
- *"satisfying outcome"* = kết quả thỏa mãn

**📝 YOUR SCORE: [ ]✅ [ ]⚠️ [ ]❌**

## 🃏 CARD 5 — Filler Phrases (Buy Thinking Time)

**The interviewer asks a hard question. You need 3-5 seconds.**

✅ Natural filler phrases:
- *"That's a great question — let me think through that for a moment..."*
- *"Hmm, let me break that down..."*
- *"That's something I've dealt with — the way I think about it is..."*
- *"So there are a few angles to this..."*
- *"Good question. So the way I understand it is..."*
- *"Let me make sure I answer that fully..."*

❌ Avoid:
- *"Umm... I think... maybe..."* (sounds unsure)
- Long silences with no bridge phrase
- *"I don't know"* (always add "but here's how I'd approach it")

**📝 YOUR SCORE: [ ]✅ [ ]⚠️ [ ]❌**

## 🃏 CARD 6 — Closing the Introduction

**Q: "Is there anything else you'd like to add about yourself?"**

✅ Model answer:
> "I think what I'd like to emphasize is that I'm not just a developer who writes code — I genuinely care about the systems I build and the teams I work with. I take ownership seriously, communicate proactively, and I love the challenge of making complex things simple.

> Outside of work, I actively study system design and architecture patterns — not because I have to, but because I find it genuinely interesting. I feel like that curiosity is what keeps me growing as an engineer.

> I'm really looking forward to learning more about this role and the team."

**🔑 Key phrases:**
- *"I'd like to emphasize"* = tôi muốn nhấn mạnh
- *"take ownership"* = chịu trách nhiệm
- *"proactively"* = chủ động
- *"genuinely"* = thực sự, thật sự

**📝 YOUR SCORE: [ ]✅ [ ]⚠️ [ ]❌**

## 🎤 Speaking Practice

**Exercise 1 — 90-Second Timer**
Record yourself for 90 seconds answering: "Tell me about yourself."
Listen back and check:
- [ ] Did you use filler phrases naturally (not "umm")?
- [ ] Did you mention relevant tech stack?
- [ ] Did you end with a connection to the role?
- [ ] Was your pace comfortable (not too fast)?

**Exercise 2 — Shadow Practice**
Read the model answers aloud 3 times, mimicking the rhythm and emphasis.
