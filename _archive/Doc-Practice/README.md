# TÀI LIỆU ĐÀO TẠO & LUYỆN PHỎNG VẤN SENIOR JAVA GAME BACKEND

Chào mừng bạn đến với lộ trình đào tạo chuyên sâu dành cho Java Game Backend Engineer. Bộ tài liệu này được thiết kế để đưa bạn từ cấp độ Junior lên Mid/Senior với tư duy thực chiến và chiều sâu kỹ thuật cao nhất.

## 🚀 Cấu trúc bài học mới
Để đảm bảo bạn nắm vững từ gốc rễ đến ngọn, tất cả các bài học trong đợt cập nhật này đều tuân thủ cấu trúc 4 bước:
1.  **Nó là gì?**: Định nghĩa lý thuyết chuẩn xác.
2.  **Dùng để làm gì?**: Ứng dụng thực tế trong Game Backend.
3.  **Dùng khi nào?**: Phân tích ngữ cảnh và các lựa chọn đánh đổi (Trade-offs).
4.  **Cách thức hoạt động (Cơ chế)**: Đi sâu vào "dưới nắp máy" của công nghệ.

## 📁 Danh sách các bài học

1.  [**PHẦN 1: JAVA CORE (NỀN TẢNG)**](Part1_JavaCore.md)
    - OOP vs Composition, Memory Model (Stack/Heap), Reflection, Java 8+ Streams.
2.  [**PHẦN 2: MULTI-THREADING (TRÁI TIM)**](Part2_MultiThreading.md)
    - Executors & ThreadPools, JMM & Visibility, Race conditions, Deadlock, Lock-free (CAS).
3.  [**PHẦN 3: NETWORK PROGRAMMING**](Part3_Network.md)
    - TCP vs UDP, Protobuf Serialization, Netty Framework, Socket internals.
4.  [**PHẦN 4: MICROSERVICES & SYSTEM DESIGN**](Part4_Microservices.md)
    - Monolith vs Microservices, Stateful vs Stateless, gRPC vs REST vs Kafka.
5.  [**PHẦN 5: KAFKA (EVENT-DRIVEN)**](Part5_Kafka.md)
    - Broker, Partition, Consumer Group, Internal Log mechanism, Zero Copy.
6.  [**PHẦN 6: REDIS (CACHE & REALTIME)**](Part6_Redis.md)
    - In-memory data structures, ZSet (Leaderboard), RDB vs AOF, Sharding.
7.  [**PHẦN 7: MONGODB / DATABASE DESIGN**](Part7_Database.md)
    - Embedding vs Referencing, Indexing internals, Shard Key strategy.
8.  [**PHẦN 8: KUBERNETES & CLOUD**](Part8_K8s_Cloud.md)
    - Docker, Kubernetes, Agones (Game Server Scaling), CI/CD flow.
9.  [**PHẦN 9: SYSTEM DESIGN (LEVEL SENIOR)**](Part9_SystemDesign_Senior.md)
    - 1M CCU, Global Chat broadcasting, Rate Limiting, High Availability.
10. [**PHẦN 10: GOLANG & KOTLIN (BONUS)**](Part10_Golang_Kotlin.md)
    - Performance Go (Goroutines) vs Kotlin (Coroutines), JVM interoperability.
11. [**PHẦN 11: LỘ TRÌNH HỌC (ROADMAP)**](Part11_Roadmap.md)
    - Lộ trình 3 tháng - 6 tháng - 1 năm - Senior Architect.
12. [**PHẦN 12: MOCK INTERVIEW**](Part12_MockInterview.md)
    - Bộ câu hỏi phỏng vấn tư duy và trải nghiệm thực tế (Junior -> Senior).
